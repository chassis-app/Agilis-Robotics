# 2026-03-14 Change Plan

This plan is based on the current app structure in `src/pages`, `src/config/navigation.ts`, `src/router.tsx`, the existing mock/store models, and the client comments below.

## Current-state summary

- The app already has screens for `Goods Receipt`, `Item Master`, `Sales Orders`, `Work Orders`, `Material Issue Notices`, `Material Issue Requests`, `BOM`, `ECR/ECO`, `NC/CAPA`, `Inspections`, `Traceability`, and `Projects`.
- The app does not currently have first-class concepts for:
  - goods return
  - complaints
  - internal QC
  - market-specific order/BOM filtering
  - formal vs informal document sequences
  - dual order types (sales vs internal)
  - BOM comparison
  - permanent BOM layer numbering
  - separate request-vs-issue execution trace for inventory

## Numbered page changes

3. Parts master should keep the current category and add support for three more category fields.
4. Inventory terminology is confirmed:
   - `領料` = request to withdraw inventory
   - `發料` = confirmation of inventory withdrawal
5. For build-order split, the working assumption is child production batches under one parent build order.
6. Dual sequences should cover the whole production process.
7. `Internal QC` should support a custom WYSIWYG template editor built inside the system.
8. `Traceability` should support entry from any step by document / part / product number and be presented as a workflow instead of a tree.
