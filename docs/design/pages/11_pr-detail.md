# Screen Prompt: Purchase Requisition Detail / Create / Edit

## Screen ID
`PROC-PR-DETAIL`

## Route
- Create: `/procurement/purchase-requisitions/new`
- Detail/Edit: `/procurement/purchase-requisitions/:id`

## Purpose
Create, view, and edit purchase requisitions. Includes line item management, real-time validation (stock check, safety stock impact), approval tracking, document links, and audit trail. The Submit action triggers the two-level approval workflow.

## Workflows
WF-01 (Requisition to PO)

## Requirements
RQ-001 (bilingual), RQ-002 (multi-currency), RQ-004 (PR before PO), RQ-005 (two-level approval + submit), RQ-020 (document links), RQ-026 (safety stock impact)

## Template
Detail Page / Form Page (design spec Sections 9.2, 9.5)

## Header Fields

| Field | Type | Required | Editable | Notes |
|---|---|---|---|---|
| PR Number | Text (mono) | Auto | Never | Auto-generated by numbering service |
| Status | Badge | Auto | Never | Colored per document status colors |
| Requester | Text | Auto | Never | Pre-filled from logged-in user session |
| Created Date | Date | Auto | Never | Set on creation |
| Required Date | Date picker | Yes | Draft only | Default: today + 14 days |
| Priority | Select | Yes | Draft only | Options: Normal, Urgent. Default: Normal |
| Notes | Textarea | No | Draft only | Max 500 characters, character counter |

## Line Item Fields

| Field | Type | Required | Notes |
|---|---|---|---|
| Item | Searchable select | Yes | Shows item_no + name (CN) + name (EN); search across both languages; shows item image thumbnail |
| Revision | Select | Yes | Filtered to released revisions of selected item only; default to latest released |
| Quantity | Number | Yes | Positive integer or decimal per item UOM; min 1 |
| UOM | Text (read-only) | Auto | Auto-filled from item master after item selection |
| Warehouse | Select | Yes | Dropdown of active warehouses for current site |
| Required Date | Date picker | Yes | Per-line required date; default from header required date |
| Estimated Unit Price | Number + currency | No | Number input with currency selector (¥ RMB / $ USD per RQ-002) |
| Line Notes | Text | No | Free text, max 200 characters |

### Line Item Interactions
- **Add Line**: Click [+ Add Line] appends empty row at bottom
- **Delete Line**: Trash icon on each row; confirmation dialog if line has data
- **Item selection**: Opens searchable dropdown with:
  - Search by item_no, name (CN), name (EN)
  - Results show: item_no | name (CN) | name (EN) | type icon | current stock
  - After selection: auto-fills UOM, shows stock info
- **Inline stock check**: After item + warehouse selected, system shows:
  - ✓ green + "N available" if stock sufficient
  - ⚠ warning + "N available" (amber) if requested qty > available but > 0
  - ✗ danger + "0 available" (red) if zero stock
  - Safety stock impact note if available qty after issue would breach safety threshold (RQ-026)

## Validation Rules

| Rule | Trigger | Display |
|---|---|---|
| Required fields | On blur + on submit | Red border + error text below field |
| Item must have released revision | On item select | Error if no released revisions exist |
| Quantity > 0 | On blur | Error text |
| At least 1 line required | On submit | Banner error at top of lines section |
| Required date not in past | On blur | Warning (allows override with note) |
| Duplicate item+revision+warehouse | On item/rev/wh change | Warning: "Duplicate line — consider merging" |

### Submit Action Flow
1. User clicks [Submit for Approval]
2. Client validates all required fields and at least 1 line
3. If validation fails: scroll to first error, show error messages
4. If validation passes: open e-signature dialog (design spec Section 10.2)
   - Meaning: "I submit this purchase requisition for approval"
   - User re-enters password
5. On success: status changes to "Submitted", toast "PR submitted for approval", page becomes read-only
6. System creates workflow_instance and sends notifications to L1 approver
